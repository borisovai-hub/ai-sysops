# Исправления скриптов установки на одну машину

## Исправленные проблемы

### 1. Нумерация этапов в install-all.sh
- **Проблема**: Неправильная нумерация этапов (1/7, 2/7, но потом 7/9, 8/9, 9/9)
- **Исправление**: Исправлена нумерация на последовательную (1/9, 2/9, ..., 9/9)

### 2. Настройка external_url в GitLab
- **Проблема**: sed может не найти строку для замены, если она уже изменена
- **Исправление**: Добавлена проверка наличия строки и два варианта замены (замена существующей или добавление новой)

### 3. Установка Node.js для n8n
- **Проблема**: apt update не выполнялся перед установкой Node.js
- **Исправление**: Добавлен apt update перед установкой Node.js

### 4. Поиск директории management-ui
- **Проблема**: Путь к management-ui мог быть неправильным при загрузке на сервер
- **Исправление**: Улучшен поиск директории management-ui с проверкой нескольких возможных расположений:
  - `~/install/management-ui` (когда скрипты в `~/install/scripts/single-machine`)
  - `../management-ui` (относительно скрипта)
  - `./management-ui` (в текущей директории)
  - И другие варианты

### 5. Установка wget для Traefik
- **Проблема**: wget может быть не установлен
- **Исправление**: Добавлена проверка и установка wget перед скачиванием Traefik

### 6. Создание директории dynamic в Traefik
- **Проблема**: Директория `/etc/traefik/dynamic` может не существовать
- **Исправление**: Добавлено создание директории перед созданием конфигурационных файлов

### 7. DNS API скрипт
- **Проблема**: Неполная реализация, зависимость от vm1-traefik
- **Исправление**: Создан полноценный скрипт `manage-dns` с поддержкой Cloudflare и DigitalOcean

### 8. Проверка конфигурации Traefik
- **Проблема**: Traefik мог запуститься с неправильной конфигурацией
- **Исправление**: Добавлена проверка наличия конфигурационного файла перед запуском

### 9. Ожидание запуска GitLab
- **Проблема**: GitLab может не успеть запуститься после reconfigure
- **Исправление**: Добавлено ожидание доступности GitLab с проверкой через curl

### 10. Улучшенная финальная проверка
- **Проблема**: Простая проверка статуса не давала достаточно информации
- **Исправление**: Добавлена детальная проверка каждого сервиса с указанием команд для диагностики

### 11. Проверка портов перед установкой
- **Проблема**: Порты 80 и 443 могли быть заняты
- **Исправление**: Добавлена проверка занятости портов с предупреждением

### 12. Обработка ошибок npm install
- **Проблема**: Ошибки npm install не обрабатывались
- **Исправление**: Добавлена проверка кода возврата npm install

### 13. Увеличение времени ожидания запуска сервисов
- **Проблема**: Сервисы могли не успеть запуститься за 3 секунды
- **Исправление**: Увеличено время ожидания до 5 секунд для n8n и Traefik

## Дополнительные улучшения

1. Все скрипты используют `set -e` для немедленного выхода при ошибке
2. Добавлены проверки наличия необходимых команд перед использованием
3. Улучшены сообщения об ошибках с указанием команд для диагностики
4. Добавлены проверки существования файлов и директорий перед использованием

## Проверка готовности

Все скрипты готовы к использованию и должны работать с первой попытки при соблюдении системных требований.
