# Настройка DNS для веб-сайта (с учётом доступности из России)

Инструкция по настройке DNS записей для корректной работы сайта с российскими пользователями.

## Типичные причины недоступности сайта из России

1. **NS-серверы за рубежом** — Cloudflare, зарубежные регистраторы могут работать нестабильно
2. **Cloudflare Proxy** — IP-адреса Cloudflare периодически блокируются Роскомнадзором
3. **Зарубежный хостинг** — некоторые IP-диапазоны заблокированы
4. **DNSSEC проблемы** — некорректная настройка может вызывать сбои
5. **Медленный DNS** — зарубежные NS отвечают медленнее для российских пользователей

---

## 1. Выбор NS-серверов (Name Servers)

NS-серверы — это серверы, которые отвечают за разрешение DNS запросов для вашего домена.

### Рекомендуемые варианты для России

| Провайдер | NS-серверы | Особенности |
|-----------|------------|-------------|
| **Reg.ru** | `ns1.reg.ru`, `ns2.reg.ru` | Российский, стабильный |
| **RU-CENTER** | `ns1.nic.ru`, `ns2.nic.ru` | Крупнейший российский регистратор |
| **Selectel DNS** | `ns1.selectel.ru`, `ns2.selectel.ru` | Быстрый, бесплатный DNS хостинг |
| **Яндекс 360** | `dns1.yandex.net`, `dns2.yandex.net` | Для доменов на Яндексе |
| **Cloudflare** | `*.ns.cloudflare.com` | Быстрый, но могут быть блокировки |
| **DigitalOcean** | `ns1.digitalocean.com` | Зарубежный, могут быть задержки |

### Рекомендация

Для максимальной доступности из России используйте **российские NS-серверы**:

```
ns1.reg.ru
ns2.reg.ru
```

или

```
ns1.selectel.ru
ns2.selectel.ru
```

### Как изменить NS-серверы

1. Войдите в панель управления регистратора домена
2. Найдите раздел "DNS" или "Name Servers"
3. Замените текущие NS на выбранные
4. Дождитесь распространения (до 48 часов, обычно 2-4 часа)

## 2. A-записи для сайта

A-записи связывают доменное имя с IP-адресом сервера.

### Базовая настройка

| Тип | Имя | Значение | TTL |
|-----|-----|----------|-----|
| A | `@` | `IP_СЕРВЕРА` | 300-3600 |
| A | `www` | `IP_СЕРВЕРА` | 300-3600 |
| A | `api` | `IP_СЕРВЕРА` | 300-3600 |

**Пример для borisovai.ru:**

| Тип | Имя | Значение | TTL |
|-----|-----|----------|-----|
| A | `@` | `123.45.67.89` | 3600 |
| A | `www` | `123.45.67.89` | 3600 |
| A | `api` | `123.45.67.89` | 3600 |

### Поддомены для Traefik/Let's Encrypt (n8n, management-ui)

Если в логах Traefik ошибка **NXDOMAIN** или *"check that a DNS record exists for this domain"* для `n8n`, `ui`, `dns` — добавьте A-записи для этих поддоменов **в каждой зоне** (borisovai.ru и borisovai.tech):

| Тип | Имя | Значение | TTL |
|-----|-----|----------|-----|
| A | `n8n` | `IP_СЕРВЕРА` | 3600 |
| A | `ui` | `IP_СЕРВЕРА` | 3600 |
| A | `dns` | `IP_СЕРВЕРА` | 3600 |

После добавления записей подождите распространения DNS (2–15 мин), затем Traefik повторно запросит сертификаты.

**Если не хотите создавать поддомены n8n/ui для всех базовых доменов:** задайте в `/etc/install-config.json` только те хосты, для которых есть DNS: `n8n_hosts` и `management_ui_hosts` (полные имена через запятую). Пример: `"n8n_hosts": "n8n.borisovai.ru"`, `"management_ui_hosts": "ui.borisovai.ru,dns.borisovai.ru,dns.borisovai.tech"`. Затем на сервере: `sudo ./configure-traefik.sh --force` и `sudo systemctl restart traefik`. Подробнее: [scripts/single-machine/DIAGNOSE_404.md](scripts/single-machine/DIAGNOSE_404.md).

### TTL (Time To Live)

- **300** (5 минут) — для частых изменений, быстрое переключение
- **3600** (1 час) — стандартное значение
- **86400** (24 часа) — для стабильных записей, меньше нагрузки на DNS

**Рекомендация:** Используйте TTL 300-600 при начальной настройке, затем увеличьте до 3600.

## 3. CNAME-записи

CNAME создаёт псевдоним для другого домена.

| Тип | Имя | Значение | TTL |
|-----|-----|----------|-----|
| CNAME | `www` | `borisovai.ru` | 3600 |

**Важно:** CNAME нельзя использовать для корневого домена (`@`). Для `@` всегда используйте A-запись.

### Альтернатива: ALIAS/ANAME

Некоторые DNS провайдеры поддерживают ALIAS записи для корневого домена:

| Тип | Имя | Значение |
|-----|-----|----------|
| ALIAS | `@` | `www.borisovai.ru` |

## 4. Настройка без Cloudflare Proxy (для России)

Если используете Cloudflare и сайт недоступен из России:

### Вариант 1: Отключить Proxy (оранжевое облако → серое)

1. Войдите в Cloudflare Dashboard
2. DNS → выберите A-запись
3. Нажмите на оранжевое облако, чтобы оно стало серым
4. Сохраните

**Результат:** Трафик идёт напрямую на сервер, минуя Cloudflare.

### Вариант 2: Перенести DNS на российский сервер

1. Создайте DNS зону на российском провайдере (Selectel, Reg.ru)
2. Перенесите все записи
3. Измените NS-серверы у регистратора

### Вариант 3: Использовать Cloudflare с российским сервером

Если сервер в России (Selectel, Yandex Cloud, VK Cloud):
- Cloudflare может работать, так как IP российский
- Но NS-серверы Cloudflare могут блокироваться

## 5. Полный пример DNS зоны для сайта

```dns
; NS записи (управляются регистратором)
@           IN  NS      ns1.reg.ru.
@           IN  NS      ns2.reg.ru.

; A записи - основной сайт
@           IN  A       123.45.67.89
www         IN  A       123.45.67.89

; A записи - API и сервисы
api         IN  A       123.45.67.89
gitlab      IN  A       123.45.67.89
mail        IN  A       123.45.67.89

; CNAME (альтернатива A для www)
; www       IN  CNAME   borisovai.ru.

; TXT записи (верификация, SPF)
@           IN  TXT     "v=spf1 a mx -all"

; CAA запись (опционально, ограничивает CA)
@           IN  CAA     0 issue "letsencrypt.org"
```

## 6. Проверка доступности из России

### Онлайн-инструменты

| Сервис | URL | Описание |
|--------|-----|----------|
| **check-host.net** | https://check-host.net/ | Проверка с разных точек мира, включая Россию |
| **2ip.ru** | https://2ip.ru/site-availability/ | Проверка доступности сайта из России |
| **host-tracker.com** | https://www.host-tracker.com/ | Мониторинг с множества локаций |
| **ping.pe** | https://ping.pe/ | Ping и traceroute из разных стран |

### Командная строка

```bash
# Проверка DNS разрешения
dig borisovai.ru @8.8.8.8          # Google DNS
dig borisovai.ru @77.88.8.8        # Яндекс DNS (Россия)
dig borisovai.ru @1.1.1.1          # Cloudflare DNS

# Сравнение результатов
echo "Google DNS:"
dig +short borisovai.ru @8.8.8.8
echo "Yandex DNS:"
dig +short borisovai.ru @77.88.8.8

# Проверка NS серверов
dig NS borisovai.ru

# Трассировка DNS
dig +trace borisovai.ru

# Проверка доступности (если есть VPN в России)
curl -I https://borisovai.ru
```

### Проверка через российские DNS

Если сайт не резолвится через российские DNS, проблема в NS-серверах:

```bash
# Яндекс DNS
dig borisovai.ru @77.88.8.8
dig borisovai.ru @77.88.8.1

# DNS от МТС
dig borisovai.ru @195.34.31.50

# DNS от Ростелеком
dig borisovai.ru @217.117.66.100
```

## 8. Диагностика проблем

### Сайт не открывается только из России

1. **Проверьте NS-серверы:**
   ```bash
   dig NS borisovai.ru
   ```
   Если NS зарубежные (cloudflare.com, digitalocean.com) — возможна блокировка.

2. **Проверьте IP-адрес:**
   ```bash
   dig A borisovai.ru @77.88.8.8
   ```
   Сравните результат с ожидаемым IP.

3. **Проверьте доступность IP из России:**
   - https://check-host.net/ → введите IP → проверьте российские точки

### DNS резолвится, но сайт не открывается

1. **Проверьте, не заблокирован ли IP:**
   - https://blocklist.rkn.gov.ru/ (реестр Роскомнадзора)

2. **Проверьте сертификат:**
   ```bash
   openssl s_client -connect borisovai.ru:443 -servername borisovai.ru
   ```

3. **Проверьте firewall на сервере:**
   ```bash
   # На сервере
   ufw status
   iptables -L -n
   ```

### Сайт открывается медленно

1. **Проверьте время DNS-запроса:**
   ```bash
   dig borisovai.ru | grep "Query time"
   ```

2. **Проверьте ping до сервера:**
   ```bash
   ping borisovai.ru
   ```
   Для России: <100ms — хорошо, >200ms — плохо.

3. **Проверьте traceroute:**
   ```bash
   traceroute borisovai.ru
   ```

### Проблемы DNS в сети (ошибки разрешения имён, неверный IP)

См. **[DNS_TROUBLESHOOTING.md](DNS_TROUBLESHOOTING.md)** — очистка кэша на ПК, конфликт порта 53 на сервере (systemd-resolved и dnsmasq), перезапуск сервисов. На сервере можно запустить: `sudo scripts/dns-api/fix-dns.sh`.

## 9. Быстрый чек-лист

### Начальная настройка DNS

- [ ] Выбрать NS-серверы (рекомендуется российские для аудитории из России)
- [ ] Создать A-запись для `@` (корневой домен)
- [ ] Создать A-запись для `www`
- [ ] Создать A-записи для поддоменов (api, mail, gitlab и т.д.)
- [ ] Установить TTL 300-600 на время настройки
- [ ] Проверить резолвинг через российские DNS (77.88.8.8)
- [ ] Проверить доступность через check-host.net
- [ ] Увеличить TTL до 3600 после стабилизации

### При проблемах с доступностью из России

- [ ] Проверить NS-серверы (не заблокированы ли)
- [ ] Отключить Cloudflare Proxy (если используется)
- [ ] Проверить IP сервера на блокировку
- [ ] Рассмотреть перенос на российский хостинг
- [ ] Перенести DNS на российский провайдер

## 10. Настройка DNS через скрипты проекта

Если настроен DNS API, можно управлять записями автоматически:

```bash
# Настройка DNS API (Cloudflare/DigitalOcean/локальный)
sudo ./setup-dns-api.sh

# Создание A записи
manage-dns create www 123.45.67.89
manage-dns create api 123.45.67.89

# Проверка
manage-dns test www
```
