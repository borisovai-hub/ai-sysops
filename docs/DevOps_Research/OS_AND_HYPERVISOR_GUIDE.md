## 🖥️ Где запускать?

### Вариант A: Физический сервер (НЕ рекомендуется для Tier 1)
```
Плюсы:
  ✅ Максимальная производительность
  ✅ Нет оверхеда гипервизора

Минусы:
  ❌ Дорого для стартапа ($300+ в месяц)
  ❌ Сложно масштабироваться
  ❌ Сложнее бэкапы/восстановление

Когда использовать: Tier 3 (если требуется ультравысокая производительность)
```

### Вариант B: Proxmox (РЕКОМЕНДУЕТСЯ для Tier 1-2)
```
Что это: Open-source гипервизор (как VMware, но бесплатный)
Работает на базе: KVM + LXC

Плюсы:
  ✅ Бесплатный
  ✅ Отличный для виртуализации
  ✅ Легко управлять VMs (веб-интерфейс)
  ✅ Встроены снимки (snapshots) для бэкапов
  ✅ Живая миграция VMs между хостами
  ✅ Отличная поддержка community

Минусы:
  ⚠️ Требует собственный сервер (не подходит облачным)

Для нас: ИДЕАЛЬНО для Tier 1-2
Tier 1:  1 Proxmox хост → 1 VM (Debian 12)
Tier 2:  1-2 Proxmox хоста → 3 VMs (HA cluster)
```

### Вариант C: Облачные VMs (DigitalOcean, Hetzner, Contabo)
```
Что это: Готовые VMs в облаке, выбираешь OS

Плюсы:
  ✅ Не нужно своё железо
  ✅ Платишь за то что используешь
  ✅ Легко масштабировать
  ✅ Не нужно техническое обслуживание железа

Минусы:
  ⚠️ Немного медленнее (сетевой оверхед)
  ⚠️ Привязка к провайдеру

Для нас: ХОРОШО для Tier 1-2
Tier 1:  1 облачный VM (6 CPU, 8GB RAM) - $50-100/мес
Tier 2:  3 облачных VM - $450-900/мес
```

### Вариант D: Kubernetes (только для Tier 3)
```
Что это: Оркестратор контейнеров (управляет Docker контейнерами)

На чём работает:
  - Managed K8s (EKS, GKE, AKS) - дорого
  - Самостоятельный K8s - сложно
  - Proxmox + K8s - хорошо

Для нас:
  Tier 1-2: НЕ используем K8s (сложновато)
  Tier 3: Используем K8s (требует мощности)
```

## 💡 Рекомендуемые комбинации

### ✅ Вариант 1: Proxmox на своём железе (ЛУЧШИЙ для Tier 1-2)

```
架构:
┌─ Proxmox (гипервизор)
│  └─ Debian 12 VM #1 (для Tier 1)
│  └─ Debian 12 VM #2 (для Tier 2)
│  └─ Debian 12 VM #3 (для Tier 2)
│
Требует:
  - Один физический сервер (6+ ядер, 16GB+ ОЗУ)
  - Собственное железо или виртуальный сервер с поддержкой вложенной виртуализации

Стоимость:
  - Сервер: $100-200/месяц (мощнее, чем для одной VM)
  - ПО: $0 (Proxmox бесплатен)

Плюсы:
  ✅ Полный контроль над VMs
  ✅ Легко делать снимки (snapshots)
  ✅ Легко масштабировать (добавить VM за минуты)
  ✅ Живая миграция VMs

Минусы:
  ❌ Нужно обслуживать Proxmox хост
  ❌ Более сложная архитектура
```

### ✅ Вариант 2: Облачные VMs (ПРОСТОЙ для Tier 1)

```
Архитектура:
┌─ DigitalOcean / Hetzner / Contabo
│  └─ Debian 12 Droplet #1 (Tier 1)
│
Стоимость:
  - VM: $50-100/месяц
  - ПО: $0

Плюсы:
  ✅ Супер простой для начинающих
  ✅ Не нужно обслуживать физический сервер
  ✅ Готово за 5 минут
  ✅ Есть бэкапы, восстановление, рейплики

Минусы:
  ❌ Привязка к облачному провайдеру
  ❌ Немного медленнее чем физический сервер
  ❌ Сложнее миграция на другой провайдер
```

### ✅ Вариант 3: Proxmox + Облако (МАСШТАБИРУЕМЫЙ для Tier 2-3)

```
Архитектура:
┌─ Proxmox на своём железе
│  └─ Debian 12 VM #1 (основной)
│
└─ Облачный VM
   └─ Debian 12 VM #2 (резервный / Tier 2)
   └─ Debian 12 VM #3 (резервный / Tier 2)

Плюсы:
  ✅ Гибкость
  ✅ Отказоустойчивость (если падает Proxmox хост)
  ✅ Расширяемость

Минусы:
  ❌ Сложнее управлять
  ❌ Дороже (платишь за оба хоста)
```

## 🎯 Рекомендация по выбору

### Для Tier 1 (Стартап)

**Вариант A (Облако)**
```
┌─ DigitalOcean / Hetzner
└─ 1x Debian 12 VM (6 CPU, 8GB)

Стоимость:        $50-100/мес
Сложность:        Минимальная
Время развёртки:  5 минут
Для кого:         Для первого прототипа
```

**Вариант B (Proxmox)**
```
┌─ Свой сервер Proxmox
└─ 1x Debian 12 VM (6 CPU, 8GB)

Стоимость:        $100-200/мес
Сложность:        Средняя (нужно поставить Proxmox)
Время развёртки:  1-2 часа
Для кого:         Если хочется полный контроль
```

### Для Tier 2 (Растущий стартап)

**Вариант A (Облако)**
```
┌─ DigitalOcean / Hetzner
├─ VM #1 - Debian 12 (API сервер)
├─ VM #2 - Debian 12 (БД)
└─ VM #3 - Debian 12 (Кэш/backup)

Стоимость:        $450-900/мес
Сложность:        Низкая
Для кого:         Быстрого роста, простоты
```

**Вариант B (Proxmox)**
```
┌─ Proxmox хост #1
├─ VM #1 - Debian 12 (API сервер)
└─ VM #2 - Debian 12 (БД)

└─ Proxmox хост #2 (резервный)
   └─ VM #3 - Debian 12 (Синхронизированная копия)

Стоимость:        $200-400/мес
Сложность:        Средняя (администрирование)
Для кого:         Желающих максимального контроля + HA
```

### Для Tier 3 (Enterprise)

```
Kubernetes кластер (3-5 нод)
├─ На Proxmox (если свои сервера)
└─ На облаке (EKS, GKE, etc.) - дороже

Debian 12 на каждой ноде
Контейнеры (Docker) внутри

Стоимость:        $1500-3500/мес
Сложность:        Высокая
Для кого:         Enterprise, большие нагрузки
```

## 🔄 Миграция: Облако → Proxmox (или наоборот)

**Если начали с облака, хотим перейти на Proxmox**:
```
1. Создать снимок VM в облаке
2. Скачать образ диска
3. Импортировать в Proxmox
4. Запустить VM в Proxmox
5. Переключить DNS

Время: 1-2 часа
Уровень сложности: Средний
```

**Если начали с Proxmox, хотим перейти в облако**:
```
1. Создать снимок VM в Proxmox
2. Экспортировать образ диска
3. Загрузить в облако
4. Запустить VM в облаке
5. Переключить DNS

Время: 1-2 часа
Уровень сложности: Средний
```

## 📋 Установка: Быстрый гайд

### Вариант 1: Облако (DigitalOcean)
```bash
1. Зайти на DigitalOcean.com
2. Create Droplet
3. Выбрать: Debian 12, 6 CPU, 8GB RAM
4. Выбрать регион (Frankfurt = низкая латенция в EU)
5. Создать
6. SSH в Droplet
7. Запустить скрипт установки:
   curl https://our-repo/install.sh | bash
```

### Вариант 2: Proxmox
```bash
1. Скачать Proxmox VE ISO (бесплатно)
2. Записать на USB
3. Загрузиться с USB на сервер
4. Установить (как Debian)
5. Открыть веб-интерфейс https://ip:8006
6. Создать VM (Debian 12)
7. Запустить скрипт установки в VM
```
