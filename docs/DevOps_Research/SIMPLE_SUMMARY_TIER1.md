# Что мы строим: Простое объяснение для Tier 1

## 🎯 Главная идея

Система, которая сама управляет вашим серверным оборудованием и приложениями. Как опытный сотрудник, который следит за всем 24/7 и исправляет проблемы без вашего участия.

---

## 📦 Для кого это?

**Tier 1** — для стартапов и небольших команд:
- Один слабый сервер (6 ядер CPU, 8GB памяти)
- Несколько микросервисов (nginx, база данных, приложение)
- Малая команда (1-3 разработчика)
- Ограниченный бюджет

---

## 🚀 Что система будет делать?

### Автоматически (без вашего участия)
✅ **Мониторить здоровье**: Каждые 5 минут проверяет, все ли сервисы работают
✅ **Перезагружать сломанное**: Если nginx упал — автоматически поднимает его
✅ **Анализировать логи**: Ищет ошибки, предупреждает вас
✅ **Масштабировать**: Если одному сервису мало ресурсов — добавляет ему
✅ **Очищать**: Удаляет старые логи и временные файлы

### С вашим одобрением
⚠️ **Обновлять приложение**: Предлагает обновить версию, вы кликаете "согласен"
⚠️ **Менять конфигурацию**: "Хотим изменить настройки", вы проверяете и одобряете
⚠️ **Расширять инфраструктуру**: "Сервер переполнен, хотим перейти на более мощный", вы решаете

## 🔧 Как это технически работает?

### Компоненты Tier 1

```
Ваш сервер (6 CPU, 8GB RAM)
│
├─ Приложения (nginx, БД, API)
├─ ИИ-агент (Claude 4) — мозг системы
├─ Terraform — хранит конфигурацию в git
├─ Vault — безопасно хранит пароли
├─ Prometheus — собирает метрики
└─ Система мониторинга — показывает графики
```

### Как они работают вместе

```
1️⃣ ИИ-агент просыпается каждые 5 минут
   └─ Спрашивает: "Какое текущее состояние системы?"

2️⃣ Собирает информацию
   ├─ Метрики (CPU, память, диск)
   ├─ Логи ошибок
   ├─ Статус каждого сервиса
   └─ Желаемое состояние (хранится в git через Terraform)

3️⃣ Анализирует
   └─ "nginx не запущен, а должен быть запущен!"

4️⃣ Принимает решение
   └─ "Перезагружу nginx"

5️⃣ Выполняет
   └─ Отправляет команду: restart nginx

6️⃣ Проверяет результат
   ├─ nginx поднялся? ✅
   ├─ Здоров? ✅
   └─ Логирует: "в 10:05 перезагрузил nginx, всё хорошо"

7️⃣ Спит до следующей проверки
```

## 🛡️ Безопасность (просто о сложном)

**Ваши пароли**
- ❌ Не хранятся в открытом виде
- ✅ Зашифрованы в файле Vault
- ✅ Доступ логируется (вы видите кто и когда брал пароль)

**Что может делать агент**
- ✅ Перезагружать сервисы (безопасно)
- ✅ Смотреть логи
- ❌ НЕ может удалять базу данных
- ❌ НЕ может менять пароли без вас
- ❌ НЕ может отправлять данные в интернет

**Если что-то сломалось**
- Агент автоматически откатывает (возвращает предыдущее состояние)
- Логирует всё в файл (можно посмотреть позже "что произошло в 3 ночи")

## ❓ Частые вопросы

**Q: Я могу доверить это системе?**
A: Да. Система не может удалить базу данных или украсть пароли. Для опасных операций нужно ваше одобрение.

**Q: Что если агент "сойдёт с ума"?**
A: Есть 7 слоёв защиты. Если агент делает странное — система автоматически откатывает изменения.

**Q: Можно ли отключить агента?**
A: Да, в любой момент кнопкой "pause" в веб-интерфейсе.

**Q: Что если упадёт интернет?**
A: Сервер продолжит работать. Агент будет работать в "автономном режиме" до восстановления связи.

**Q: Где данные моих пользователей?**
A: На вашем серверке (вы его арендуете). Никуда не отправляется кроме запросов к Claude API.
