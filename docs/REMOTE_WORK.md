# Работа с сервером через Remote-SSH и агенты

Агенты Cursor/VS Code могут работать с удалённой машиной, если открыть проект на сервере через Remote-SSH.

## Как это работает

Агенты выполняют команды в **активном терминале** и работают с **текущим workspace**. Если workspace — папка на сервере, открытая через Remote-SSH:

- команды из агента выполняются на сервере;
- чтение и правка файлов идут на удалённой файловой системе.

Контекст агента в этом случае — контекст удалённой машины.

## Что нужно сделать

### 1. Подключиться по Remote-SSH и открыть папку на сервере

- **F1** → «Remote-SSH: Connect to Host…» → выбрать хост (или ввести `user@host`).
- После подключения: **File → Open Folder** → выбрать каталог на сервере (например `/root/install` или каталог, куда загружены скрипты).
- Workspace должен быть именно эта удалённая папка, а не локальный каталог проекта.

### 2. Убедиться, что терминал — удалённый

После открытия папки на хосте новый терминал в Cursor/VS Code по умолчанию открывается на сервере. Агент будет выполнять команды в нём.

### 3. SSH без пароля (по ключу)

Чтобы агент и расширение стабильно работали с сервером, настройте вход по ключу и при необходимости SSH config (алиас хоста, ключ, пользователь).

- Настройка SSH ключа: см. [UPLOAD_EXAMPLES.md](UPLOAD_EXAMPLES.md) (раздел «Настройка SSH ключа»).
- Пример SSH config для одной машины: [config/single-machine/ssh-config.example](config/single-machine/ssh-config.example). Скопируйте нужный блок в `~/.ssh/config` (Windows: `%USERPROFILE%\.ssh\config`) и подставьте свой IP, пользователя и путь к ключу.

## Управление подключением и работой агента настройками проекта

Подключение к серверу и папка, в которой работает агент, задаются в репозитории:

| Файл | Назначение |
|------|------------|
| [borisovai-admin-remote.code-workspace](borisovai-admin-remote.code-workspace) | Workspace с удалённой папкой. При открытии этого файла (File → Open Workspace from File…) Cursor/VS Code подключается к хосту `server` и открывает `/root/install` на сервере — агент сразу работает в контексте сервера. |
| [config/single-machine/ssh-config.example](config/single-machine/ssh-config.example) | Пример блока для `~/.ssh/config`. Хост в SSH config должен называться так же, как в `uri` workspace (сейчас `server`). |

**Как пользоваться:** скопируйте блок из `ssh-config.example` в свой `~/.ssh/config`, подставьте свой IP, пользователя и ключ. Затем откройте `borisovai-admin-remote.code-workspace` — после подключения терминал и агент будут на сервере.

**Если путь на сервере другой** (не `/root/install`): отредактируйте в `.code-workspace` поле `uri`, например `vscode-remote://ssh-remote+server/root/install` → `vscode-remote://ssh-remote+server/другой/путь`.

**Если хост в SSH config другой** (не `server`): замените в `.code-workspace` имя хоста в `uri` и в ключе `remote.SSH.remotePlatform` на своё имя из `~/.ssh/config`.

## Важно

- Если открыта **локальная** папка (например `c:\projects\borisovai-admin`), агент работает с **локальной** машиной и локальным терминалом.
- Чтобы агент работал с **удалённой** машиной, откройте workspace `borisovai-admin-remote.code-workspace` или вручную подключитесь по Remote-SSH и откройте папку на сервере.

## Итог

Агенты смогут работать с удалённой машиной при условии, что вы подключены по Remote-SSH и открыли папку на сервере как текущий workspace.
