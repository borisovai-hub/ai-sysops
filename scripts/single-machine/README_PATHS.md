# Определение расположения скриптов

Все скрипты в этой директории автоматически определяют свое расположение и могут быть запущены из любой директории.

## Как это работает

Каждый скрипт использует следующий код для определения своей директории:

```bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
```

Это позволяет:
- Запускать скрипты из любой директории
- Использовать абсолютные пути для поиска других файлов
- Не зависеть от текущей рабочей директории

## Структура путей

### install-all.sh
- Определяет `SCRIPT_DIR` - директорию со скриптами
- Определяет `INSTALL_ROOT` - корневую директорию установки (родительская директория скриптов)
- Ожидаемая структура:
  ```
  ~/install/
  ├── scripts/
  │   └── single-machine/
  │       ├── install-all.sh
  │       └── ...
  └── management-ui/
      ├── server.js
      └── ...
  ```

### install-management-ui.sh
- Определяет `SCRIPT_DIR` - директорию скрипта
- Определяет `INSTALL_ROOT` - корневую директорию установки (передается как аргумент или определяется автоматически)
- Ищет `management-ui` в следующих местах:
  1. `$INSTALL_ROOT/management-ui` (основной вариант)
  2. `$(dirname "$INSTALL_ROOT")/management-ui`
  3. `$(dirname "$SCRIPT_DIR")/management-ui`
  4. `./management-ui` (текущая директория)
  5. `../management-ui` (родительская директория)

## Примеры использования

### Запуск из директории скриптов
```bash
cd ~/install/scripts/single-machine
sudo ./install-all.sh
```

### Запуск из другой директории
```bash
cd /tmp
sudo ~/install/scripts/single-machine/install-all.sh
```

### Запуск install-management-ui.sh с указанием пути
```bash
sudo ~/install/scripts/single-machine/install-management-ui.sh /custom/path/to/install
```

## Важно

- Все пути преобразуются в абсолютные
- Скрипты не зависят от текущей рабочей директории
- Относительные пути используются только для поиска файлов в нескольких возможных местах
