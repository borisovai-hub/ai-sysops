# Настройка Proxmox VE 7 для системы управления сервисами

Подробное руководство по настройке Proxmox VE 7 для развертывания GitLab, Traefik и системы управления инструментами.

## Важно: Одна физическая машина

**Это руководство адаптировано для одной физической машины.**

Если у вас одна физическая машина:
- ✅ Используйте **один сетевой интерфейс** и мост `vmbr0`
- ✅ Все VM находятся в **одной подсети** (например, `192.168.1.0/24`)
- ✅ VM общаются между собой **напрямую** через локальную сеть
- ✅ **Firewall** на VM 2 (GitLab) ограничивает доступ только с IP VM 1 (Traefik)
- ✅ Traefik проксирует запросы к GitLab по локальному IP

**Пример конфигурации для одной физической машины:**
- Proxmox: `192.168.1.100/24`
- VM 1 (Traefik): `192.168.1.101/24` - имеет внешний доступ
- VM 2 (GitLab): `192.168.1.102/24` - доступен только из локальной сети

## Установка Proxmox VE 7

### Требования к оборудованию

**Для одной физической машины:**
- **CPU**: 64-bit процессор с поддержкой виртуализации (Intel VT-x или AMD-V)
  - Минимум 4 ядра (2 для Proxmox + 2 для VM 1 + 4 для VM 2)
  - Рекомендуется 8+ ядер для комфортной работы
- **RAM**: минимум 8GB (рекомендуется 16GB+)
  - Proxmox: ~2GB
  - VM 1 (Traefik): 2GB минимум, 4GB рекомендуется
  - VM 2 (GitLab): 4GB минимум, 8GB рекомендуется
  - Резерв: 2GB для системы
  - **Итого**: минимум 8GB, рекомендуется 16GB+
- **Диск**: минимум 100GB свободного места (рекомендуется SSD)
  - Proxmox система: ~20GB
  - ISO образы: 10-20GB
  - VM 1 (Traefik): 20GB минимум, 40GB рекомендуется
  - VM 2 (GitLab): 50GB минимум, 100GB рекомендуется
  - Снимки и бэкапы: +20-30GB
  - **Итого**: минимум 100GB, рекомендуется 200GB+
- **Сеть**: 1 сетевой интерфейс (достаточно для одной физической машины)
  - Все VM используют один мост `vmbr0`
  - VM находятся в одной подсети
  - Внешний доступ только через Traefik

3. Запишите образ на USB-накопитель или DVD

- **Ubuntu 20.04 LTS ISO**: https://releases.ubuntu.com/20.04/ubuntu-20.04.6-live-server-amd64.iso

**Рекомендация**: Используйте Debian 12 или Ubuntu 22.04 LTS для лучшей совместимости.

### Шаг 2: Установка Proxmox VE

1. **Загрузка с установочного носителя**
   - Вставьте USB/DVD с образом Proxmox
   - Загрузите сервер с этого носителя
   - Выберите "Install Proxmox VE"

2. **Настройка диска**
   - Выберите целевой диск для установки
   - Выберите файловую систему:
     - **ext4** - стандартная, простая в использовании (рекомендуется для начинающих)
     - **zfs** - продвинутая файловая система с поддержкой снимков и дедупликации (требует больше RAM)
   - Подтвердите форматирование диска
   
   **Важно**: 
   - Для ZFS рекомендуется минимум 8GB RAM
   - Для ext4 достаточно 4GB RAM
   - ZFS обеспечивает лучшую производительность и функции резервного копирования
   
   **Подробнее о настройке диска см. раздел "Настройка диска при установке Proxmox" ниже**

3. **Настройка региона**
   - Выберите страну и часовой пояс
   - Настройте раскладку клавиатуры

4. **Настройка пароля и сети**
   - Установите пароль для пользователя `root`
   - Настройте сетевой интерфейс:
     - **Hostname**: например, `proxmox.example.com`
     - **IP Address**: статический IP адрес сервера
     - **Netmask**: например, `255.255.255.0`
     - **Gateway**: IP адрес шлюза
     - **DNS Server**: IP адрес DNS сервера

5. **Завершение установки**
   - Проверьте все настройки
   - Нажмите "Install" и дождитесь завершения
   - После перезагрузки откройте веб-интерфейс: `https://<IP>:8006`

### Шаг 3: Первоначальная настройка

1. **Вход в веб-интерфейс**
   - Откройте браузер: `https://<IP_PROXMOX>:8006`
   - Войдите как `root` с паролем, установленным при инсталляции
   - Примите предупреждение о самоподписанном сертификате

2. **Обновление системы**
   ```bash

   apt update
   apt upgrade -y
   ```

3. **Настройка репозиториев (если нужно)**
   - По умолчанию используется бесплатный репозиторий
   - Для коммерческой поддержки настройте подписку

## Настройка сети

**ВАЖНО**: Если у вас одна физическая машина, используйте **Вариант 1** (один сетевой интерфейс). Внутренняя сеть между VM будет работать через виртуальный мост на одном физическом интерфейсе.

### Вариант 1: Один сетевой интерфейс (рекомендуется для одной физической машины)

Если у вас один сетевой интерфейс (одна физическая машина), Proxmox автоматически настроит мост `vmbr0`. Все VM будут использовать этот мост, но могут общаться между собой через внутреннюю сеть.

**Проверка текущей конфигурации:**
```bash
cat /etc/network/interfaces
```

**Типичная конфигурация:**
```
auto lo
iface lo inet loopback

auto enp3s0
iface enp3s0 inet manual

auto vmbr0
iface vmbr0 inet static
    address 192.168.1.100/24
    gateway 192.168.1.1
    bridge-ports enp3s0
    bridge-stp off
    bridge-fd 0
```

### Вариант 2: Два сетевых интерфейса (рекомендуется для продакшена)

**Схема:**
- `vmbr0` - внешняя сеть (публичный IP)
- `vmbr1` - внутренняя сеть (для связи между VM)

**Настройка через веб-интерфейс:**

1. Перейдите в **Datacenter → Node → System → Network**
2. Нажмите **Create → Linux Bridge**
3. Настройте первый мост (`vmbr0`):
   - **Name**: `vmbr0`
   - **Bridge ports**: выберите первый сетевой интерфейс
   - **IPv4/CIDR**: внешний IP адрес (например, `203.0.113.10/24`)
   - **Gateway**: IP адрес шлюза
   - Нажмите **Create**

4. Настройте второй мост (`vmbr1`) для внутренней сети:
   - **Name**: `vmbr1`
   - **Bridge ports**: выберите второй сетевой интерфейс
   - **IPv4/CIDR**: внутренний IP (например, `10.0.0.1/24`)
   - **Gateway**: оставьте пустым
   - Нажмите **Create**

**Или через командную строку:**

Отредактируйте `/etc/network/interfaces`:
```
auto lo
iface lo inet loopback

# Внешний интерфейс
auto enp3s0
iface enp3s0 inet manual

auto vmbr0
iface vmbr0 inet static
    address 203.0.113.10/24
    gateway 203.0.113.1
    bridge-ports enp3s0
    bridge-stp off
    bridge-fd 0

# Внутренний интерфейс
auto enp4s0
iface enp4s0 inet manual

auto vmbr1
iface vmbr1 inet static
    address 10.0.0.1/24
    bridge-ports enp4s0
    bridge-stp off
    bridge-fd 0
```

Примените изменения:
```bash
ifreload -a
```

## Создание виртуальных машин

### VM 1: Traefik Proxy

#### Шаг 1: Создание VM

1. В веб-интерфейсе Proxmox нажмите **Create VM** (правый верхний угол)
2. **General**:
   - **VM ID**: `100` (или следующий доступный)
   - **Name**: `traefik-proxy`
   - Нажмите **Next**

3. **OS**:
   - **Use CD/DVD disc image file (iso)**: выберите образ Debian 11/12 или Ubuntu 22.04
   - **Guest OS**: Linux, версия 5.x - 2.6 Kernel
   - Нажмите **Next**

4. **System**:
   - **Graphic card**: Default (VGA)
   - **Machine**: Default (i440fx)
   - **BIOS**: SeaBIOS
   - **SCSI controller**: VirtIO SCSI single
   - Нажмите **Next**

5. **Hard Disk**:
   - **Bus/Device**: VirtIO Block (рекомендуется для лучшей производительности)
   - **Storage**: выберите хранилище:
     - `local-lvm` - для LVM хранилища (рекомендуется)
     - `local-zfs` - для ZFS хранилища
     - `local` - для Directory хранилища (не рекомендуется для дисков VM)
   - **Disk size**: `20GB` (минимум для Traefik)
   - **Cache**: 
     - **Write back** - для SSD (лучшая производительность)
     - **Write through** - для HDD или если нужна надежность
     - **No cache** - отключение кэширования (не рекомендуется)
   - **Discard**: включите для SSD (TRIM поддержка)
   - **SSD emulation**: включите если используете SSD
   - Нажмите **Next**
   
   **Рекомендации по размеру диска:**
   - Traefik VM: минимум 20GB, рекомендуется 40GB
   - GitLab VM: минимум 50GB, рекомендуется 100GB+
   - Учитывайте место для логов, обновлений и данных

6. **CPU**:
   - **Sockets**: `1`
   - **Cores**: `2` (минимум)
   - **Type**: `host` (рекомендуется) или `kvm64`
   - Нажмите **Next**

7. **Memory**:
   - **Memory**: `2048` MB (минимум, рекомендуется 4096)
   - Нажмите **Next**

8. **Network**:
   - **Bridge**: `vmbr0` (внешняя сеть)
   - **Model**: VirtIO (paravirtualized)
   - **MAC Address**: оставьте автоматический или задайте вручную
   - **VLAN tag**: оставьте пустым (если не используете VLAN)
   - Нажмите **Next**

9. **Confirm**:
   - Проверьте все настройки
   - Нажмите **Finish**

#### Шаг 2: Дополнительная настройка сети (опционально)

**Для одной физической машины**: VM могут общаться между собой через один мост `vmbr0`, используя разные IP адреса в одной подсети. Дополнительный сетевой интерфейс не обязателен.

Если хотите изолировать внутреннюю сеть (опционально):

1. Выберите созданную VM → **Hardware → Add → Network Device**
2. Настройте второй сетевой интерфейс:
   - **Bridge**: `vmbr0` (тот же мост, но с другим IP)
   - **Model**: VirtIO
   - Нажмите **Add**
   
**Альтернатива**: Используйте один сетевой интерфейс с разными IP адресами в одной подсети для всех VM.

#### Шаг 3: Установка ОС

1. Выберите VM → **Console**
2. Нажмите **Start** для запуска VM
3. Установите Debian/Ubuntu:
   - Выберите язык и регион
   - Настройте сеть:
     - Для `vmbr0`: получите IP через DHCP или настройте статический
     - Для `vmbr1` (если есть): настройте статический IP (например, `10.0.0.10/24`)
   - Создайте пользователя и установите пароль
   - Завершите установку

4. **Запишите IP адреса**:
   - Внешний IP: `203.0.113.10` (пример)
   - Внутренний IP: `10.0.0.10` (пример)

### VM 2: GitLab Server

#### Шаг 1: Создание VM

Повторите шаги из VM 1, но с другими параметрами:

1. **General**:
   - **VM ID**: `101`
   - **Name**: `gitlab-server`

2. **Hard Disk**:
   - **Disk size**: `50GB` (минимум, рекомендуется 100GB)

3. **CPU**:
   - **Cores**: `4` (минимум)

4. **Memory**:
   - **Memory**: `4096` MB (минимум, рекомендуется 8192)

5. **Network**:
   - **Bridge**: `vmbr0` (для одной физической машины используйте тот же мост)
   - **Model**: VirtIO
   
   **ВАЖНО для одной физической машины**: 
   - GitLab использует тот же мост `vmbr0`, что и Traefik
   - Настройте firewall на VM 2, чтобы GitLab был доступен только из VM 1
   - Используйте разные IP адреса в одной подсети (например, `192.168.1.101` и `192.168.1.102`)

#### Шаг 2: Установка ОС

1. Установите Debian/Ubuntu аналогично VM 1
2. Настройте сеть:
   - **Для одной физической машины**: настройте статический IP в той же подсети
     - Пример: `192.168.1.102/24` (в той же подсети, что и VM 1)
     - Gateway: IP шлюза вашей сети (например, `192.168.1.1`)
     - DNS: используйте те же DNS, что и Proxmox (или публичные: 8.8.8.8, 1.1.1.1)
   - **Важно**: GitLab должен быть доступен из VM 1 (Traefik) по этому IP

3. **Запишите IP адрес**: `192.168.1.102` (пример)
   - Этот IP будет использоваться в скрипте установки Traefik для проксирования GitLab

## Настройка диска при установке Proxmox

### Выбор файловой системы

При установке Proxmox VE вам будет предложено выбрать файловую систему для диска:

#### Вариант 1: ext4 (рекомендуется для начинающих)

**Преимущества:**
- Простая в использовании
- Низкие требования к RAM (минимум 4GB)
- Стабильная и проверенная
- Хорошая производительность для большинства задач

**Недостатки:**
- Нет встроенной поддержки снимков
- Нет дедупликации
- Ограниченные возможности резервного копирования

**Когда использовать:**
- Небольшие установки
- Ограниченные ресурсы RAM
- Простые требования к резервному копированию

#### Вариант 2: ZFS (рекомендуется для продвинутых пользователей)

**Преимущества:**
- Встроенная поддержка снимков (snapshots)
- Дедупликация данных
- Автоматическая проверка целостности данных
- Отличная производительность для виртуализации
- Клонирование VM за секунды

**Недостатки:**
- Высокие требования к RAM (минимум 8GB, рекомендуется 16GB+)
- Более сложная настройка
- Требует больше дискового пространства для метаданных

**Когда использовать:**
- Достаточно RAM (16GB+)
- Нужны снимки и клонирование
- Важна целостность данных
- Несколько дисков для RAID

### Рекомендации по размеру диска

**Минимальные требования:**
- Для установки Proxmox: 32GB
- Для хранения ISO образов: +20GB
- Для VM: зависит от количества и размера VM

**Рекомендуемые размеры:**
- **Малые установки** (1-5 VM): 100-200GB
- **Средние установки** (5-20 VM): 500GB-1TB
- **Большие установки** (20+ VM): 1TB+

**Распределение пространства:**
- Proxmox система: ~20GB
- ISO образы: 20-50GB
- Снимки и бэкапы: 20-30% от размера VM
- VM диски: остальное пространство

### Настройка диска после установки

#### Проверка текущей конфигурации

```bash
# Проверка файловой системы
df -h

# Проверка разделов
lsblk

# Для ext4/LVM
vgs
lvs
pvs

# Для ZFS
zpool list
zfs list
```

#### Расширение диска (если нужно)

**Для LVM (ext4):**
```bash
# 1. Расширьте физический раздел (если добавили новый диск)
pvcreate /dev/sdb
vgextend pve /dev/sdb

# 2. Расширьте logical volume
lvextend -l +100%FREE /dev/pve/data

# 3. Расширьте файловую систему
resize2fs /dev/pve/data
```

**Для ZFS:**
```bash
# Добавление диска в пул
zpool add tank /dev/sdb

# Автоматическое расширение
zpool set autoexpand=on tank
```

## Настройка хранилищ

### Понимание типов хранилищ в Proxmox

Proxmox использует несколько типов хранилищ:

1. **local** (Directory)
   - Хранилище для ISO образов, снимков, шаблонов
   - Обычно находится в `/var/lib/vz`
   - Подходит для небольших файлов

2. **local-lvm** (LVM-Thin)
   - Хранилище для дисков виртуальных машин
   - Использует LVM thin provisioning
   - Эффективное использование дискового пространства

3. **ZFS**
   - Если выбрали ZFS при установке
   - Предоставляет снимки, клонирование, дедупликацию
   - Требует больше RAM

### Проверка доступных хранилищ

1. Перейдите в **Datacenter → Storage**
2. Проверьте наличие хранилищ:
   - `local` - для ISO образов, снимков, шаблонов
   - `local-lvm` - для дисков VM (если используется LVM)
   - `local-zfs` - для дисков VM (если используется ZFS)

### Загрузка ISO образов

#### Способ 1: Через веб-интерфейс

1. Перейдите в **Datacenter → local → ISO Images**
2. Нажмите **Upload**
3. Выберите ISO файл с вашего компьютера
4. Дождитесь завершения загрузки

#### Способ 2: Через командную строку (SCP)

```bash
# На вашем компьютере
scp debian-12.x.x-amd64-netinst.iso root@<PROXMOX_IP>:/var/lib/vz/template/iso/

# Или через веб-интерфейс: Datacenter → local → ISO Images → Upload
```

#### Способ 3: Прямая загрузка на сервер

```bash
# На Proxmox сервере
cd /var/lib/vz/template/iso/

# Загрузка Debian 12
wget https://cdimage.debian.org/debian-cd/current/amd64/iso-cd/debian-12.x.x-amd64-netinst.iso
```

## Настройка внутренней сети

### Для одной физической машины (рекомендуется)

**Схема сети:**
- Proxmox: `192.168.1.100/24` (пример)
- VM 1 (Traefik): `192.168.1.101/24` - имеет внешний доступ
- VM 2 (GitLab): `192.168.1.102/24` - в той же подсети

**Как это работает:**
- Все VM подключены к одному мосту `vmbr0`
- VM находятся в одной подсети и могут общаться напрямую
- Traefik проксирует запросы к GitLab по локальному IP
- GitLab не должен быть доступен извне напрямую (настройте firewall)

**Настройка firewall на VM 2 (GitLab):**
```bash
# На VM 2 разрешите доступ только с IP Traefik VM
ufw allow from 192.168.1.101 to any port 80
ufw deny 80
ufw enable
```

### Вариант 1: Использование отдельного моста (vmbr1) - только если есть два физических интерфейса

Если настроили `vmbr1` (требует два физических сетевых интерфейса):
- VM 1 подключена к `vmbr0` (внешняя) и `vmbr1` (внутренняя)
- VM 2 подключена только к `vmbr1` (внутренняя)
- VM могут общаться через внутреннюю сеть

**Для одной физической машины этот вариант не требуется!**

### Вариант 2: Использование одного моста с VLAN - для изоляции (опционально)

Если хотите изолировать внутреннюю сеть на одном физическом интерфейсе:

1. В настройках VM → Network:
   - **VLAN tag**: `10` (для внутренней сети)
   - VM 1: без VLAN (внешняя) и VLAN 10 (внутренняя)
   - VM 2: только VLAN 10 (внутренняя)
   
**Примечание**: Для одной физической машины это не обязательно - можно использовать одну подсеть с правильной настройкой firewall.

## Рекомендации по безопасности

### 1. Настройка Firewall в Proxmox

1. Перейдите в **Datacenter → Firewall**
2. Включите Firewall: **Options → Firewall → Enable**
3. Настройте правила:
   - Разрешите SSH (порт 22)
   - Разрешите веб-интерфейс (порт 8006)
   - Ограничьте доступ по IP при возможности

### 2. Настройка SSH

```bash
# Отредактируйте /etc/ssh/sshd_config
nano /etc/ssh/sshd_config

# Рекомендуемые настройки:
PermitRootLogin no
PasswordAuthentication no  # Используйте ключи
PubkeyAuthentication yes

# Перезапустите SSH
systemctl restart sshd
```

### 3. Регулярные обновления

```bash
# Настройте автоматические обновления безопасности
apt install unattended-upgrades
dpkg-reconfigure -plow unattended-upgrades
```

### 4. Резервное копирование

1. Перейдите в **Datacenter → Backup**
2. Настройте расписание резервного копирования:
   - **Storage**: выберите хранилище для бэкапов
   - **Schedule**: настройте расписание
   - **Compression**: включите сжатие
   - **Mode**: Snapshot (для VM)

### 5. Мониторинг

1. Настройте уведомления:
   - **Datacenter → Notifications**
   - Добавьте email для уведомлений

2. Установите мониторинг ресурсов:
   - Используйте встроенные графики в Proxmox
   - Или установите внешний мониторинг (Zabbix, Prometheus)
